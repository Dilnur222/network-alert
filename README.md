# Система оповещений о доступности сети

Автоматизированный сервис для мониторинга доступности сетевых устройств. Скрипт периодически проверяет (пингует) список устройств и в случае их недоступности:
1. Отправляет уведомление в Telegram.
2. Записывает событие в локальный лог-файл (`alerts.log`).
3. Выводит предупреждение в консоль.

## 🚀 Как запустить

### 1. Подготовка окружения
Убедитесь, что у вас установлен Python (версия 3.6+). Затем установите необходимые зависимости, открыв терминал в папке проекта:
```bash
pip install requests python-dotenv
```

### 2. Настройка конфигурации
В корне проекта должен лежать файл `.env`. Если вы скачали проект с GitHub, создайте файл `.env` и добавьте туда токены вашего Telegram-бота:
```env
TELEGRAM_TOKEN=1234567890:ABCdefGhIJKlmNoPQRstuVWXyz
TELEGRAM_CHAT_ID=123456789
```

Отредактируйте список устройств в файле `devices.json`, указав имена и IP-адреса оборудования, которое нужно мониторить.

### 3. Запуск
Запустите скрипт через терминал:
```bash
python net_alert.py
```

## 📊 Примеры данных

### Пример входных данных (`devices.json`)
```json
{
  "devices": [
    {
      "name": "my phone",
      "ip": "192.168.0.18"
    },
    {
      "name": "my pc",
      "ip": "127.0.0.1"
    }
  ]
}
```

### Пример выходных данных (Консоль)
```text
Запуск системы мониторинга...
✅ my phone (192.168.0.18) доступен
✅ my pc (127.0.0.1) доступен
------------------------------
⚠️ ALERT: my phone (192.168.0.18) недоступен!
✅ my pc (127.0.0.1) доступен
------------------------------
```

### Пример выходных данных (Лог-файл `alerts.log`)
```text
2026-05-15 03:07:05 — ⚠️ ALERT: my phone (192.168.0.18) недоступен!
2026-05-15 03:07:10 — ⚠️ ALERT: my phone (192.168.0.18) недоступен!
```
